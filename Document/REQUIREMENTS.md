# TODOリストアプリ 要件定義書

## 1. アプリ概要
シンプルで使いやすい3画面構成のTODOリストアプリケーション。
タスクの作成、編集、アーカイブ、およびアーカイブ済みタスクの閲覧機能を提供する。

## 2. 画面構成と機能

### 2.1 ホーム画面 (Home)
アプリ起動時に表示されるメイン画面。

*   **TODOリスト表示**:
    *   未完了（かつ未アーカイブ）のTODOアイテムをリスト形式で表示する。
    *   各アイテムにはタイトルを表示する。
*   **新規作成 (FAB)**:
    *   画面右下のフローティングアクションボタン (FAB) をタップすると、新規TODO作成ダイアログが表示される。
    *   ダイアログでタイトルを入力し「追加」するとリストに追加される。
*   **アーカイブ (スワイプ)**:
    *   リストアイテムを横にスワイプすると、そのアイテムは「アーカイブ」状態となり、ホーム画面のリストから非表示になる。
*   **詳細・編集 (タップ)**:
    *   リストアイテムをタップすると、そのアイテムの「詳細・編集画面」へ遷移する。
*   **アーカイブ一覧への遷移**:
    *   AppBar（画面上部）にあるアーカイブメニューアイコンをタップすると、「アーカイブ画面」へ遷移する。

### 2.2 アーカイブ画面 (Archive)
完了または保留となったタスクを確認する画面。

*   **アーカイブリスト表示**:
    *   アーカイブされたTODOアイテムをリスト形式で表示する。
    *   （要件には明記されていないが、通常は読み取り専用、または詳細画面への遷移が可能であることが望ましい。今回は詳細画面への遷移を想定）

### 2.3 詳細・編集画面 (Detail/Edit)
TODOアイテムの内容詳細を確認・編集する画面。

*   **詳細表示**:
    *   選択されたTODOアイテムのタイトルや詳細情報を表示する。
*   **編集機能**:
    *   タイトルや詳細情報の編集が可能。
    *   保存ボタン（または自動保存）で変更を確定する。

## 3. データモデル (ToDo Item)

TODOアイテムは以下の属性を持つ。

| 属性名 (論理名) | 物理名 (例) | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| **ID** | `id` | UUID / String | 一意の識別子 |
| **タイトル** | `title` | String | タスクの件名 |
| **詳細** | `description` | String | タスクの詳細内容 (任意) |
| **アーカイブフラグ** | `isArchived` | Boolean | アーカイブされているか (`true`: アーカイブ済み) |
| **作成日時** | `createdAt` | Long / Instant | 作成された日時 |
